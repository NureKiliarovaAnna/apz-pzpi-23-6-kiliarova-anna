3 ОПИС ВИКОНАНОЇ РОБОТИ
        3.1 Загальні відомості про патерни
Патерни проєктування — це типові рішення поширених проблем при проєктуванні програмного забезпечення. Вони забезпечують повторне використання перевірених архітектурних підходів.
Класифікація GoF:
* породжувальні,
* структурні,
* поведінкові.
3.2 Опис патерна
Назва: Фабричний метод (Factory Method)
Тип: породжувальний
Призначення: визначає інтерфейс для створення об’єкта, але дозволяє підкласам змінювати тип об’єкта, що створюється.
Передумови застосування:
* коли клас не знає, які саме об'єкти потрібно створювати,
* коли потрібно делегувати створення підкласам.
Переваги:
* слабке зв’язування,
* розширюваність,
* відповідність принципу відкритості/закритості.
Недоліки:
* ускладнення структури коду,
* збільшення кількості класів.
3.3 Структура патерна
Основні елементи:
* Product (інтерфейс),
* ConcreteProduct,
* Creator (фабричний метод),
* ConcreteCreator.
Взаємодія: Creator викликає factoryMethod(), який перевизначається у підкласах.

3.4 Реалізація
Опис реалізації мовою програмування: Java. Код міститься в додатку В.
Основні етапи реалізації:
1. Створення базових інтерфейсів: на першому етапі визначається інтерфейс Product, який описує загальну поведінку створюваних об’єктів. Усі конкретні продукти повинні реалізовувати цей інтерфейс;
2. Реалізація конкретних класів: створюються класи ConcreteProductA та ConcreteProductB, які реалізують інтерфейс Product. Кожен із них визначає власну реалізацію методів;
3. Створення абстрактного класу-творця: абстрактний клас Creator містить фабричний метод factoryMethod(), який повертає об’єкт типу Product. Також може містити бізнес-логіку;
4. Реалізація конкретних творців: класи ConcreteCreatorA та ConcreteCreatorB перевизначають метод factoryMethod() та створюють конкретні об’єкти продуктів;
5. Демонстрація використання: у клієнтському коді створюється об’єкт конкретного творця. Виклик методу someOperation() призводить до створення продукту через factoryMethod().

В.1 Приклад коду до патерну Фабричний метод (Factory Method)
1  interface Product {
2      void use();
3  }
4
5  class ConcreteProductA implements Product {
6      public void use() {
7          System.out.println("Product A");
8      }
9  }
10
11 class ConcreteProductB implements Product {
12     public void use() {
13         System.out.println("Product B");
14     }
15 }
16
17 abstract class Creator {
18     abstract Product factoryMethod();
19
20     void someOperation() {
21         Product product = factoryMethod();
22         product.use();
23     }
24 }
25
26 class ConcreteCreatorA extends Creator {
27     Product factoryMethod() {
28         return new ConcreteProductA();
29     }
30 }
31
32 class ConcreteCreatorB extends Creator {
33     Product factoryMethod() {
34         return new ConcreteProductB();
35     }
36 }
37
38 public class Main {
39     public static void main(String[] args) {
40         Creator creator = new ConcreteCreatorA();
41         creator.someOperation();
42     }
43 }
